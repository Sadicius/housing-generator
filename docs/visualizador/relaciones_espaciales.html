"Condicional" ya no es un valor de este catálogo — es un marcador de una regla aparte (acceso de baño según nº de baños) que depende del Program real, no se puede expresar como par de tipos. La preferencia de planta tampoco está implementada: el sistema genera una sola planta.

Selección independiente por planta: todas las estancias empiezan inactivas (mismo estilo, sin distinguir "principal" de "respaldo") — actívalas según lo que necesites en tu proyecto concreto. Bajo cubierta y semisótano incluyen aquí estancias adicionales a las de niveles_plantas.md (dormitorios/baño en bajo cubierta, despacho en semisótano) — uso práctico de esta herramienta, no una corrección del documento normativo. CORRIDOR aparece en todas las plantas (no tiene nivel propio, existe en cada una que tenga la vivienda).

Para generar con esto: tras exportar, python -m housing_generator.interface.cli.main --import-seleccion seleccion_plantas.json --output edificio.json — genera un edificio real con el generador Python (docs/GUIA_USO.md tiene más detalle, incluido cómo usarlo como librería).

El campo de área se pone en rojo si el valor declarado no alcanza el mínimo real de Tabla 1 (estancias, según su puesto por tamaño) o Tabla 2 (servicios, según subtipo) para el número TOTAL de estancias seleccionadas en todas las plantas -- se recalcula en vivo cada vez que añades o quitas una estancia, no es un valor fijo. Pasa el ratón por encima del campo para ver el mínimo exacto exigido.
